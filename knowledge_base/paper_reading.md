# 文献调研与论文阅读

## 一、文献调研的价值

文献调研是科研工作的基础环节，主要包括文献检索、论文筛选、全文阅读、版本核对、引用追踪和知识整理。

### 1. 为什么要检索文献

文献检索可以帮助研究者：

- 了解研究问题已经发展到什么程度
- 建立研究领域中的标准术语
- 梳理不同方法和论文之间的关系
- 找到常用数据集、评价指标和基线模型
- 了解该领域常见的实验规范
- 避免重复研究已经解决的问题
- 发现现有研究仍未解决的问题

文献检索的核心目标不是简单增加论文数量，而是围绕研究问题建立研究地图。

### 2. 为什么要阅读文献

论文阅读可以帮助研究者：

- 学习如何清晰地提出研究问题
- 学习如何设计实验并验证方法
- 判断研究结论是否得到证据支持
- 获取可靠的数据、方法和复现线索
- 建立规范的科研表达方式
- 发现论文的方法优势和局限性

论文的发表等级、引用量和作者影响力只能作为辅助信息，最终仍需要根据论文内容进行判断。

---

## 二、论文类型与版本

### 1. 会议论文

会议论文是在学术会议论文集中正式发表的论文。

核对会议论文时，需要检查：

- 论文题目
- 作者
- 会议名称
- 发表年份
- 正式论文集
- 页码
- DOI

计算机领域的许多重要研究成果会首先发表在学术会议中。

### 2. 期刊论文

期刊论文是在学术期刊中正式发表的论文。

核对期刊论文时，需要检查：

- 论文题目
- 作者
- 期刊名称
- 发表年份
- 卷号
- 期号
- 页码
- DOI

### 3. 预印本

预印本是正式发表前公开的论文版本，常见平台包括 arXiv。

核对预印本时，需要检查：

- 预印本编号
- 版本号
- 首次提交日期
- 最近修改日期
- 是否已经被会议或期刊录用
- 是否存在正式发表版本

arXiv 是预印本平台，论文出现在 arXiv 上不代表已经被会议或期刊正式录用。

### 4. 正式发表版本

正式发表版本通常可以通过以下来源确认：

- 会议论文集
- 期刊官方网站
- 出版社页面
- DOI 页面
- DBLP 书目页面

引用论文时，应优先使用正式发表版本的书目信息。

### 5. 论文版本关系

同一项研究可能存在多个版本：

1. 作者投稿前或投稿阶段的预印本
2. 会议正式发表版本
3. 后续补充实验形成的期刊扩展版本

不同版本可能存在以下差异：

- 题目发生变化
- 作者名单发生变化
- 实验规模增加
- 方法细节补充
- 结论发生调整
- 图表和数据发生更新

阅读和引用前必须明确当前使用的是哪个版本。

## 三、论文发表载体的等级判断

### 1. CCF 推荐目录

CCF 推荐目录主要用于评价计算机领域的学术会议和期刊。

常见等级包括：

- A 类
- B 类
- C 类

使用 CCF 推荐目录时，应注意以下问题：

- CCF 等级评价的是会议或期刊，不是单篇论文
- 高等级会议中的论文不一定与当前研究问题相关
- 低等级会议中的论文也可能包含有价值的方法
- CCF 等级只能用于辅助筛选
- 查询时需要记录目录版本和查询日期
- 最终判断仍需回到论文的问题、方法、实验和结论

### 2. 中科院分区

中科院分区主要面向学术期刊，常见表达包括：

- 大类分区
- 小类分区
- 一区
- 二区
- 三区
- 四区

使用中科院分区时，需要标明：

- 使用的分区体系
- 查询年份
- 大类分区或小类分区

### 3. JCR 分区

JCR 分区由 Clarivate 发布，常见表达包括：

- JIF
- Q1
- Q2
- Q3
- Q4

使用 JCR 分区时，需要标明：

- 查询年份
- 所属学科类别
- 该期刊在对应学科中的分区

### 4. 中科院分区与 JCR 分区的区别

| 比较项 | 中科院分区 | JCR 分区 |
|---|---|---|
| 发布来源 | 中国科学院文献情报中心 | Clarivate |
| 主要对象 | 学术期刊 | 学术期刊 |
| 常见表达 | 大类分区、小类分区 | JIF、Q1-Q4 |
| 使用要求 | 标明分区体系和年份 | 标明学科类别和年份 |

需要注意：

- 中科院一区和 JCR Q1 不是完全相同的概念
- 期刊分区不能用于给学术会议划分 A、B、C 等级
- 期刊分区不能代替对论文内容的判断
- 发表载体等级不等于单篇论文质量

## 四、研究问题与检索问题

文献检索前应先明确研究问题，避免直接使用过于宽泛的关键词。

需要回答以下问题：

- 研究对象是什么
- 需要完成什么任务
- 关注什么方法或学习机制
- 需要解决什么具体问题
- 希望找到哪一类文献
- 文献在当前研究中承担什么作用

一个可执行的检索问题可以表述为：

> 我想了解某一研究任务中，某类方法如何解决某个具体问题。

例如：

> 我想了解序列推荐任务中，基于自注意力的方法如何缓解长期依赖问题。

---

## 五、关键词矩阵

关键词矩阵可以从多个维度构建。

| 维度 | 主要内容 |
|---|---|
| 研究对象 | 所属领域、研究对象或应用场景 |
| 研究任务 | 模型需要完成的预测或推荐任务 |
| 研究方法 | 使用的模型、机制或算法 |
| 数据场景 | 数据类型、反馈形式和交互场景 |
| 具体问题 | 当前研究需要解决的困难 |
| 文献角色 | 综述、基准、复现或代表性论文 |

关键词需要同时准备中文概念和英文表达。

进行英文论文检索时，通常优先使用英文关键词。

## 六、检索式的迭代

文献检索不是一次完成的，需要根据检索结果不断调整关键词。

### 1. 初始检索

首先使用研究对象或研究方向进行宽泛检索。

示例：

```text
sequential recommendation
```

这一阶段主要用于了解研究领域的整体范围。

### 2. 寻找综述论文

在研究对象后加入 `survey` 或 `review`。

示例：

```text
"sequential recommendation" survey
```

综述论文可以帮助了解：

- 研究任务分类
- 主要技术路线
- 常用数据集
- 评价指标
- 代表性论文
- 现有研究问题

### 3. 加入方法关键词

在研究对象后加入具体方法。

示例：

```text
"sequential recommendation" self-attention
```

### 4. 加入具体问题

在研究对象和方法后加入需要解决的问题。

示例：

```text
"sequential recommendation" self-attention "long-term dependency"
```

### 5. 使用模型名称检索

找到代表性模型后，可以直接使用模型名称进行检索。

示例：

```text
SASRec sequential recommendation
```

### 6. 扩展后续研究

围绕种子论文继续检索：

```text
SASRec improvement
SASRec reproduction
SASRec benchmark
SASRec related work
```

## 七、检索过程记录

每轮检索都应记录：

- 检索日期
- 检索平台
- 使用的关键词
- 关键词组合方式
- 检索结果的主要方向
- 结果是否过宽或过窄
- 增加、删除或替换关键词的原因
- 下一轮检索计划

推荐使用以下模板：

```text
检索日期：

检索平台：

研究问题：

本轮检索式：

检索结果概况：

存在的问题：

关键词调整原因：

下一轮检索式：
```

保留检索记录可以提高文献调研过程的可复现性。

## 八、Google Scholar 的使用

Google Scholar 适合进行广泛的学术文献检索。

主要用途包括：

- 搜索论文题目和作者
- 使用关键词发现论文
- 查看论文发表年份
- 查找论文 PDF 来源
- 查看被引用次数
- 查看引用该论文的后续研究
- 查找相关论文
- 发现同一论文的不同版本

使用 Google Scholar 时需要注意：

- 引用量不能单独证明论文质量
- 右侧 PDF 可能来自作者主页或其他页面
- 搜索结果中的版本不一定是正式发表版本
- 作者、年份和发表载体需要进一步核对

---

## 九、DBLP 的使用

DBLP 主要用于核对计算机领域论文的书目信息。

可以核对：

- 论文题目
- 作者
- 会议或期刊
- 发表年份
- 卷号
- 期号
- 页码
- DOI
- BibTeX
- 作者的其他论文

DBLP 的优势是书目信息相对规范，但它不能代替对论文内容的阅读和判断。

推荐流程为：

```text
Google Scholar 发现论文
→ DBLP 核对书目信息
→ DOI 或出版社页面确认正式版本
→ 获取论文全文
→ 阅读和筛选
```

## 十、引用关系追踪

以一篇代表性论文为种子论文，可以从三个方向扩展文献。

### 1. 向后追踪

查看种子论文的参考文献，寻找：

- 研究问题的来源
- 经典理论
- 早期代表性方法
- 基础模型
- 常用实验设置

### 2. 向前追踪

查看引用种子论文的后续研究，寻找：

- 改进方法
- 复现研究
- 对比实验
- 局限性分析
- 后续扩展工作

### 3. 横向追踪

查找与种子论文同期或相近的研究，包括：

- 同类方法
- 相邻研究任务
- 相同数据集上的实验
- 同一作者的其他论文
- 与种子论文进行比较的工作

引用关系追踪的目标是建立论文之间的联系，而不是无限扩大论文数量。

---

## 十一、论文全文来源与版本核对

获取论文全文时，推荐按照以下顺序查找：

1. 出版社页面或 DOI 页面
2. 会议或期刊官方网站
3. arXiv 作者稿
4. 作者主页或项目主页
5. 学校图书馆或机构订阅

版本核对需要检查：

- 论文题目是否一致
- 作者名单是否一致
- 发表年份是否一致
- 发表载体是否明确
- 当前 PDF 是否为最终版本
- DOI 是否正确
- 引用时应该使用哪个版本

能够下载论文不代表论文已经正式发表。

来源不明确的 PDF 不能直接作为最终版本使用。

## 十二、常见检索偏差与修正

| 常见偏差 | 具体表现 | 修正方法 |
|---|---|---|
| 结果过宽 | 论文数量过多，研究方向偏离 | 增加任务词、方法词、问题词或年份 |
| 结果过窄 | 检索结果很少或没有结果 | 删除限制词，使用同义词或上位词 |
| 只找到预印本 | 无法确认正式发表信息 | 使用 DBLP、DOI 或出版社页面核对 |
| 只看引用量 | 高引用论文与当前问题无关 | 阅读摘要、方法、实验和结论 |
| 只看发表等级 | 忽略论文实际内容 | 判断研究相关性和证据完整性 |
| 只依赖插件 | 插件结果可能滞后或匹配错误 | 返回官方目录和原始来源核验 |
| 只使用当前关键词 | 遗漏同义表达和相邻方向 | 补充同义词、缩写和上位词 |

## 十三、十分钟快速筛选法

论文初步筛选可以按照以下时间分配完成。

| 时间 | 查看内容 | 需要回答的问题 |
|---|---|---|
| 1 分钟 | 元数据 | 论文身份和版本是否明确 |
| 2 分钟 | 摘要 | 研究任务、方法和主要结论是什么 |
| 3 分钟 | 引言与结论 | 为什么开展研究，与已有方法有什么区别 |
| 2 分钟 | 模型图和结果表 | 作者使用什么证据支持结论 |
| 2 分钟 | 代码、数据和项目页 | 是否具有复现或后续研究价值 |

快速筛选需要回答三个核心问题：

1. 这篇论文研究什么
2. 它与当前研究问题有什么关系
3. 它是否值得进入精读列表

快速筛选不能代替完整精读。

## 十四、论文分类

### 1. 精读

适用于以下论文：

- 与当前研究问题高度相关
- 方法和实验设计清晰
- 可以作为核心参考文献
- 可以作为实验基线
- 对后续研究有直接作用

### 2. 备查

适用于以下论文：

- 可以补充研究背景
- 提供数据集或评价指标信息
- 与当前研究方向相关
- 当前优先级不高
- 后续可能需要再次查阅

### 3. 排除

适用于以下论文：

- 研究任务不符合
- 方法与当前问题关系较弱
- 版本或来源不明确
- 实验信息不完整
- 距离当前研究目标较远

论文筛选结果表示其在当前研究任务中的优先级，不代表对论文永久性的质量评价。

## 十五、论文精读内容

精读一篇论文时，应重点记录以下内容。

### 1. 基本信息

- 论文题目
- 作者
- 发表年份
- 发表载体
- DOI
- 论文版本

### 2. 研究问题

- 论文解决什么问题
- 该问题为什么重要
- 现有方法存在哪些不足

### 3. 核心方法

- 方法的总体结构
- 关键模块
- 输入与输出
- 训练目标
- 与已有方法的区别

### 4. 实验设计

- 使用的数据集
- 数据划分方式
- 基线模型
- 评价指标
- 参数设置
- 消融实验
- 敏感性分析

### 5. 实验结果

- 主要结果
- 与基线模型的比较
- 不同数据集上的表现
- 实验结果是否支持论文结论

### 6. 论文评价

- 主要贡献
- 方法优势
- 存在的局限性
- 复现难点
- 与当前研究的关系

## 十七、文献调研标准流程

1. 明确研究对象、研究任务和具体问题
2. 建立中英文关键词矩阵
3. 使用宽泛关键词寻找研究入口
4. 搜索综述论文了解领域整体情况
5. 加入任务词、方法词和问题词缩小范围
6. 找到代表性论文并建立种子论文
7. 向后追踪参考文献
8. 向前追踪引用论文
9. 横向查找相关研究
10. 使用 Google Scholar 发现论文
11. 使用 DBLP 核对书目信息
12. 使用 DOI 或出版社页面确认正式版本
13. 使用十分钟快速筛选法分类论文
14. 将论文分为精读、备查和排除
15. 使用 Zotero 管理论文和 PDF
16. 使用 Obsidian 记录论文笔记
17. 保留检索过程、筛选依据和版本信息

---

## 十八、核心原则

- 检索文献是为了建立研究地图
- 阅读文献是为了学习可靠的科研方法
- 发表等级不等于单篇论文质量
- 引用量不能单独证明论文价值
- 能够下载不等于已经正式发表
- 论文版本必须经过核对
- 检索过程需要保留记录
- 论文筛选需要围绕当前研究问题
- 工具可以提高效率，但不能代替研究判断
- 重要事实和结论必须返回原始来源核验
