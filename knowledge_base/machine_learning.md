
# 机器学习与深度学习基础

## 一、常见机器学习任务

研究问题决定学习任务，学习任务决定模型输出、损失函数和评价指标。

| 任务 | 模型输出 | 常见应用 |
|---|---|---|
| 回归 | 连续数值 | 价格预测、评分预测 |
| 分类 | 离散类别或类别概率 | 点击预测、风险识别 |
| 排序 | 对多个候选对象排序 | 搜索排序、推荐排序 |
| 序列预测 | 下一时刻或下一事件 | 行为序列预测 |
| 生成 | 新的文本或内容 | 文本生成、推荐解释 |

## 二、监督学习基本组成

监督学习通常包括以下要素：

| 概念 | 含义 |
|---|---|
| 样本 | 模型处理的一条数据 |
| 特征 | 输入模型的信息 |
| 标签 | 模型需要预测的真实结果 |
| 模型 | 从输入到输出的映射关系 |
| 参数 | 模型通过训练学习的数值 |
| 损失函数 | 衡量预测与真实结果的差异 |
| 优化器 | 根据梯度更新模型参数 |
| 评价指标 | 衡量模型任务表现 |

## 三、模型训练闭环

机器学习模型的基本训练过程为：

```text
读取数据
→ 划分Batch
→ 前向传播
→ 得到模型输出
→ 计算Loss
→ 反向传播
→ 计算梯度
→ 更新参数
→ 验证模型
```

## 四、参数与超参数

### 1. 参数

参数由模型在训练过程中自动学习，例如：

- 权重
- 偏置
- Embedding向量
- 神经网络中的权重矩阵

### 2. 超参数

超参数由实验者提前设置，例如：

- 学习率
- Batch Size
- Epoch
- 网络层数
- 隐藏层维度
- 正则化系数

参数由训练过程学习，超参数需要通过经验或验证集选择。

## 五、训练集、验证集和测试集

| 数据集 | 主要作用 | 是否更新参数 |
|---|---|---:|
| 训练集 | 学习模型参数 | 是 |
| 验证集 | 调整超参数、选择模型 | 否 |
| 测试集 | 最终独立评价 | 否 |

测试集不能参与模型训练和方案选择，否则会造成数据泄漏。

## 六、Batch与Epoch

### 1. Batch

Batch表示一次输入模型的一组样本。

使用Batch训练可以：

- 降低一次计算的内存消耗
- 提高计算效率
- 使模型在一个Epoch中多次更新参数
- 在训练效率和梯度稳定性之间取得平衡

### 2. Epoch

Epoch表示模型完整遍历一次训练集。

一个Epoch中的Batch数量通常为：

```text
训练样本数量 ÷ Batch Size
```

当无法整除时，最后一个Batch的样本数可能小于设定的Batch Size。

## 七、前向传播与反向传播

### 1. 前向传播

前向传播是输入数据经过模型得到预测结果的过程。

```text
输入特征 → 模型计算 → 模型输出
```

### 2. 反向传播

反向传播根据Loss计算每个参数的梯度。

```python
loss.backward()
```

### 3. 参数更新

优化器根据梯度更新模型参数。

```python
optimizer.step()
```

一次标准更新流程为：

```python
optimizer.zero_grad()
loss.backward()
optimizer.step()
```

## 八、Logit与概率

Logit是模型经过概率转换之前的原始输出。

二分类任务中，可以使用Sigmoid函数将Logit转换为概率：

```python
probability = torch.sigmoid(logit)
```

多分类任务中，通常使用Softmax将多个Logit转换为类别概率。

Logit不是概率，它可以取任意实数。概率通常位于0到1之间。

## 九、Loss与评价指标

### 1. Loss

Loss用于衡量模型预测结果与真实标签之间的差异，并为参数更新提供方向。

常见损失函数：

| 任务 | 常见损失函数 |
|---|---|
| 回归 | MSE、MAE |
| 二分类 | Binary Cross Entropy |
| 多分类 | Cross Entropy |
| 排序 | Pairwise Loss、Listwise Loss |

### 2. 评价指标

评价指标用于从具体任务角度衡量模型表现。

常见分类指标：

- Accuracy
- Precision
- Recall
- F1-score
- AUC

Loss用于训练，评价指标用于解释结果，二者作用不同。

## 十、欠拟合、过拟合与泛化

### 1. 欠拟合

模型没有充分学习数据中的规律，训练集和验证集表现都较差。

### 2. 过拟合

模型过度适应训练数据，训练集表现较好，但验证集表现下降。

### 3. 泛化能力

泛化能力是模型在未参与训练的新数据上保持有效表现的能力。

判断过拟合时，可以同时观察训练集与验证集Loss：

- 二者都较高：可能欠拟合
- 训练Loss下降，验证Loss上升：可能过拟合
- 二者整体下降且差距较小：训练状态通常较合理

## 十一、常见模型

| 模型 | 核心思想 | 适用场景 |
|---|---|---|
| 线性模型 | 对输入特征加权求和 | 简单分类和回归 |
| 决策树 | 根据条件递归划分数据 | 结构化数据 |
| MLP | 多层线性变换与非线性激活 | 非线性特征组合 |
| CNN | 提取局部空间模式 | 图像、局部特征 |
| RNN | 按顺序传递隐藏状态 | 序列数据 |
| LSTM | 使用门控机制保留长期信息 | 长序列建模 |
| Transformer | 使用Attention建模全局关系 | 文本和长序列 |

## 十二、线性分类模型

线性分类模型对输入特征进行加权求和：

```text
输出 = 特征 × 权重 + 偏置
```

PyTorch中的线性层表示为：

```python
model = nn.Linear(input_dim, output_dim)
```

线性模型结构简单、容易解释，但只能直接学习线性决策边界。

## 十三、Dataset与DataLoader

### 1. Dataset

Dataset负责组织样本，并提供按索引读取数据的能力。

### 2. DataLoader

DataLoader负责：

- 按Batch读取数据
- 打乱训练数据
- 控制Batch Size
- 支持并行加载

```python
loader = DataLoader(
    dataset,
    batch_size=batch_size,
    shuffle=True
)
```

训练集通常需要打乱，验证集和测试集通常不需要打乱。

## 十四、训练模式与评价模式

训练时使用：

```python
model.train()
```

验证和测试时使用：

```python
model.eval()
```

评价阶段通常还需要关闭梯度记录：

```python
with torch.no_grad():
    predictions = model(features)
```

这样可以减少内存消耗，并避免无意中更新模型参数。

## 十五、模型保存与最佳模型选择

训练多个Epoch时，可以根据验证集表现保存最佳模型。

```python
best_state = deepcopy(model.state_dict())
```

训练结束后恢复最佳参数：

```python
model.load_state_dict(best_state)
```

最佳模型应由验证集选择，而不是根据测试集选择。

## 十六、模型训练检查清单

判断模型训练流程是否正确，可以检查：

1. 输入特征和标签Shape是否匹配
2. 模型输出与标签Shape是否匹配
3. Loss是否为标量
4. 训练前是否清空梯度
5. 是否执行反向传播
6. 模型参数是否发生变化
7. 验证阶段是否关闭梯度
8. 训练集和验证集Loss是否符合预期
9. 测试集是否只用于最终评价
