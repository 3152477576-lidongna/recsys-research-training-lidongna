
## 一、课程主题

本次课程围绕科研文献调研展开，主要学习了文献类型与版本识别、会议和期刊等级判断、检索关键词设计、检索结果筛选、引用关系追踪、Zotero 文献管理、科研工具选择、实验记录、版本管理以及智能体辅助科研的使用边界。

---

## 二、文献调研的价值

### 1. 为什么要检索文献

文献检索的主要作用包括：

- 了解研究问题已经发展到什么程度
- 建立标准术语、方法路线和论文之间的关系
- 找到常用数据集、评价指标、基线模型和实验规范
- 避免重复研究已经解决的问题
- 发现现有研究尚未解决的问题，为后续选题提供依据

检索文献的核心目标是建立研究地图，而不是简单收集大量论文。

### 2. 为什么要阅读文献

阅读文献可以帮助研究者：

- 学习如何清晰、可验证地提出研究问题
- 学习如何使用实验和证据支持研究结论
- 获得可靠的方法、数据与复现实验线索
- 建立研究标准，减少表达模糊和结论错误

高质量文献不能只看发表等级，还需要判断论文与研究问题的相关性、来源可靠性、研究方法和证据是否充分。

## 三、论文类型、版本与发表等级

### 1. 常见论文类型

#### 会议论文

核对内容包括：

- 会议名称
- 发表年份
- 正式论文集
- DOI 信息

#### 期刊论文

核对内容包括：

- 期刊名称
- 卷号与期号
- 页码
- 发表年份
- DOI 信息

#### 预印本

预印本通常发布在 arXiv 等平台，需要核对：

- 版本号
- 提交日期
- 后续是否正式发表

arXiv 只是预印本平台，不能直接说明论文已经被会议或期刊录用。

#### 正式发表版本

引用时应优先使用会议论文集、期刊页面或出版社提供的正式版本信息。

### 2. 论文版本关系

论文可能依次存在以下版本：

1. 作者投稿前或投稿阶段的预印本
2. 会议正式发表版本
3. 后续扩展形成的期刊版本

不同版本的题目、实验内容和结论可能发生变化，因此阅读和引用前需要进行版本核对。

### 3. CCF 推荐目录

CCF 推荐目录主要用于判断计算机领域会议和期刊的发表载体等级。

- A 类、B 类和 C 类代表会议或期刊的推荐等级
- CCF 等级不等于单篇论文的实际质量
- 高等级会议中的论文也可能与当前研究问题无关
- CCF 等级和论文引用量只能用于辅助筛选
- 最终仍需阅读论文的问题、方法、证据和复现条件

使用 CCF 目录时，需要记录目录版本和查询日期，防止目录更新造成判断差异。

### 4. 中科院分区与 JCR 分区

中科院分区和 JCR 分区都主要面向学术期刊，但二者来源和划分方式不同。

| 比较项 | 中科院分区 | JCR 分区 |
|---|---|---|
| 发布来源 | 中国科学院文献情报中心 | Clarivate |
| 主要对象 | 学术期刊 | 学术期刊 |
| 常见表达 | 大类分区、小类分区 | JIF、学科分区 Q1-Q4 |
| 使用要求 | 标明分区体系和版本年份 | 标明学科类别和年份 |

需要注意：

- 不能把中科院一区和 JCR Q1 当作完全相同的概念
- 期刊分区不能直接用于给计算机会议划分 A、B、C 等级
- 期刊等级不能替代对单篇论文内容的判断

## 四、推荐系统常见会议与期刊

推荐系统研究涉及多个研究方向，不同发表载体关注的问题也不同。

| 研究类型 | 常见发表载体 | 主要研究内容 |
|---|---|---|
| 推荐系统与信息检索 | RecSys、SIGIR、CIKM、WSDM | 推荐、检索、搜索和用户建模 |
| Web、数据挖掘与人工智能 | WWW、KDD、AAAI、IJCAI | Web 智能、数据挖掘与人工智能方法 |
| 通用机器学习 | NeurIPS、ICML、ICLR | 表示学习、优化和生成模型 |
| 常见期刊 | TOIS、TKDE、TIST | 信息系统、知识与数据工程、智能系统 |

不同会议和期刊覆盖的研究方向与评价标准不同，不能把所有载体简单归为同一等级。

## 五、从研究方向卡片构建检索问题

文献检索前需要先回答以下问题：

- 研究对象是什么
- 需要完成什么任务
- 关注什么学习机制或方法
- 需要解决什么具体问题
- 文献在研究中承担什么作用

一个完整的检索问题可以表示为：

> 我想了解某一任务中，某类方法如何解决某个具体问题。

检索前还需要明确希望找到的文献类型，例如综述论文、代表性论文、基准论文或复现论文。

### 1. 序列推荐关键词矩阵

| 维度 | 中文概念 | 英文检索词 |
|---|---|---|
| 研究对象 | 序列推荐 | sequential recommendation、sequential recommender system、POI |
| 研究任务 | 下一物品预测 | next-item prediction、next-item recommendation |
| 研究方法 | 自注意力、Transformer | self-attention、Transformer-based recommendation、LLM、contrastive learning |
| 数据场景 | 用户行为序列、隐式反馈 | user behavior sequence、implicit feedback |
| 具体问题 | 长期依赖、数据稀疏、流行度偏差 | long-term dependency、data sparsity、popularity bias |
| 文献角色 | 综述、基准、复现 | survey、review、benchmark、reproducibility |

### 2. 多方向关键词扩展

不同推荐系统方向可以继续扩展关键词。

#### 图推荐

- graph collaborative filtering
- graph recommendation
- GNN-based recommendation
- oversmoothing
- sparsity

#### 多模态推荐

- multimodal recommendation
- visual recommendation
- modality fusion
- modality alignment
- missing modality

#### 大语言模型推荐

- LLM for recommendation
- generative recommendation
- instruction tuning
- hallucination
- efficiency

还可以加入对比学习、频域方法、扩散模型和生成式检索等横向技术路线。

检索词通常由“研究对象、学习机制、研究任务和具体问题”组合形成。

## 六、检索式的迭代方法

文献检索不是一次完成的，需要根据结果不断调整关键词。

### 1. 检索迭代流程

1. 使用宽泛关键词进行初步检索  
   例如：`sequential recommendation`

2. 加入 `survey` 或 `review` 寻找综述论文  
   例如：`sequential recommendation survey`

3. 加入方法词缩小范围  
   例如：`sequential recommendation self-attention`

4. 使用代表性模型名称定位种子论文  
   例如：`SASRec sequential recommendation`

5. 通过引用关系和相关论文继续扩展  
   例如：`SASRec cited by`、`SASRec related work`

### 2. 检索记录内容

每轮检索都应记录：

- 使用了哪些关键词
- 为什么增加、删除或替换关键词
- 检索结果是否过宽或过窄
- 检索结果偏向了哪个方向
- 下一轮准备如何调整

通过保留检索过程，可以提高文献调研的可复现性。

## 七、Google Scholar 与 DBLP 的使用

### 1. Google Scholar

Google Scholar 适合：

- 广泛搜索论文题目、作者和相关信息
- 查看论文发表年份和 PDF 来源
- 追踪被引用论文
- 查看相关论文
- 发现论文的多个版本

使用时应注意：

- 引用量不能单独证明论文质量
- 右侧 PDF 可能来自作者主页或其他非正式页面
- 需要继续核对论文的正式发表来源

### 2. 引用关系追踪

以一篇种子论文为中心，可以从三个方向扩展。

#### 向后追踪

查看种子论文的参考文献，寻找：

- 问题来源
- 经典方法
- 理论基础
- 早期代表性工作

#### 向前追踪

查看引用该论文的后续研究，寻找：

- 改进方法
- 复现研究
- 对比实验
- 批评与局限性分析

#### 横向追踪

查找同时期的相关方法、相邻任务以及同一作者的其他工作。

引用关系的作用是理解论文之间的联系，不是无限扩展文献数量。

### 3. DBLP

DBLP 适合核对：

- 作者信息
- 会议或期刊名称
- 发表年份
- 卷期与页码
- DOI
- BibTeX 信息
- 同一作者的其他论文

DBLP 可以核对论文书目信息，但不能代替对论文内容的阅读和判断。

### 4. Google Scholar 与 DBLP 的分工

- 使用 Google Scholar 广泛发现论文和引用关系
- 使用 DBLP 核对作者、载体、年份和 BibTeX
- 使用 DOI 页面或出版社页面确认正式发表版本
- 完成核对后再阅读论文全文

## 八、论文版本与全文来源核对

获取论文全文时，推荐按照以下顺序查找：

1. 出版社页面或 DOI 页面
2. arXiv 作者稿
3. 作者主页或项目主页
4. 学校图书馆或机构订阅

核对论文版本时需要检查：

- 题目是否发生变化
- 作者是否发生变化
- 发表年份是否一致
- 发表载体是否一致
- 当前 PDF 是否为最终版本
- 引用时应使用哪个版本的信息

“能够下载论文”不等于“论文已经正式发表”。

## 九、常见检索偏差与修正方法

| 常见偏差 | 具体表现 | 修正方法 |
|---|---|---|
| 检索结果过宽 | 出现大量论文，研究方向偏离 | 增加任务词、方法词、问题词和年份 |
| 检索结果过窄 | 只有少量论文或没有结果 | 删除过强限制，替换同义词或使用上位词 |
| 只找到预印本 | 无法确认正式发表信息 | 使用 DBLP、DOI 或出版社页面核验 |
| 只看分区和引用量 | 论文与当前研究问题无关 | 回到摘要、方法和实验判断相关性 |
| 依赖插件判断 | 插件数据可能滞后或解析错误 | 返回原始来源进行核对 |

## 十、论文快速筛选方法

可以使用十分钟快速筛选法判断论文是否值得进一步阅读。

| 时间 | 查看内容 | 需要回答的问题 |
|---|---|---|
| 1 分钟 | 元数据 | 论文身份和版本是否明确 |
| 2 分钟 | 摘要 | 研究任务、方法和主要结论是什么 |
| 3 分钟 | 引言与结论 | 为什么研究，与已有方法有什么区别 |
| 2 分钟 | 模型图和结果表 | 作者使用什么证据支持结论 |
| 2 分钟 | 代码、数据和项目页 | 是否具有复现价值或后续研究价值 |

快速筛选只需要回答三个核心问题：

1. 这篇论文研究什么
2. 与当前研究问题有什么关系
3. 是否值得进入精读列表

### 论文筛选分类

#### 精读

论文与当前问题高度相关，方法和证据清晰，可以作为后续重点阅读对象。

#### 备查

论文只需要记录其中的背景、数据集或方法信息，目前不是最高优先级。

#### 排除

论文任务不符合、版本不明确或距离当前研究目标较远。

筛选结果代表当前研究任务下的优先级，不代表论文永久性的质量评价。

## 十一、Zotero 文献管理

### 1. 基本操作

使用 Zotero 可以完成：

- 按研究方向创建分类
- 从网页导入文献条目
- 检查并修改题目、作者、年份和发表载体
- 关联论文 PDF
- 添加标签、状态和用途说明

### 2. 最小归档标准

每篇论文至少需要完成：

- 归入一个研究分类
- 添加一个明确标签
- 写一句保留、精读或排除的理由

例如：

> 标签：序列推荐、基准模型、待复现  
> 归档理由：该论文提出序列推荐经典模型，可作为后续实验基线。

### 3. 同步、备份与隐私

#### 同步

- 使用个人账号同步文献条目
- 确认附件是否包含在同步方案中

#### 备份

- 重要 PDF、笔记和导出文件应保留本地副本
- 同步成功不能替代独立备份

#### 隐私

- 截图时不要暴露账号、项目名称和私人信息
- 未公开论文、审稿材料和实验资料不能随意上传
- 上传前需要确认材料是否具有公开权限

## 十二、引文导出与阅读笔记

### 1. Zotero 内置导出

Zotero 可以导出 BibTeX、RIS 等格式，适合一次性导出参考文献。

导出前需要检查：

- 论文题目
- 作者
- 年份
- 发表载体
- DOI

### 2. Better BibTeX

Better BibTeX 适合长期使用 LaTeX 写作的场景，可以：

- 维护自动更新的 `.bib` 文件
- 固定引用键
- 支持持续写作和参考文献管理

选择建议：

- 偶尔导出文献时使用 Zotero 内置导出
- 长期使用 LaTeX 写作时使用 Better BibTeX

引文格式正确的前提是文献条目信息正确。

### 3. Markdown 与 Obsidian

Markdown 适合轻量记录、版本管理和跨工具分享。

Obsidian 可以作为本地知识库，用于连接论文、概念、实验和研究问题。

推荐采用“一篇论文对应一篇笔记”的方式，统一记录：

- 论文研究任务
- 采用的方法
- 数据集和评价指标
- 实验结果
- 与当前研究的关系
- 尚未理解的问题
- 后续复现计划

记录时应先保证内容准确，再考虑链接、标签和复杂知识图谱。

## 十三、科研工具的选择

### 1. 在线笔记协作

#### Notion

适合：

- 数据库式资料整理
- 统一视图管理
- 共享页面
- 小组协作任务

#### 语雀

适合：

- 中文在线知识库
- 文档共享
- 团队课程协作
- 国内网络环境下的资料整理

出现共享需求时，可以使用在线工具，但仍需保留本地备份。

未公开论文、实验数据和敏感信息应先确认权限，不能直接同步到在线平台。

### 2. 写作与排版

#### Overleaf

适合在线编辑 LaTeX 文档，便于多人协作、使用模板、管理公式和参考文献。

#### LaTeX

适合结构化论文排版、公式编写、图表引用和文献管理，可以通过编译日志定位格式错误。

推荐先使用小模板测试正文、图片和参考文献，再迁移到完整论文模板，避免一次性出现大量编译问题。

### 3. 公式与文字识别

SimpleTex 等工具可以识别：

- 数学公式
- 图片文字
- 表格内容

识别后需要人工核对：

- 上下标
- 希腊字母
- 矩阵符号
- 括号
- 分式
- 对齐方式
- 编号

公式识别工具只能辅助录入，不能代替对公式含义和实验结果的理解。

### 4. 截图与图示

- PixPin 适合截图、长截图和标注
- draw.io 适合流程图、系统结构图和论文方法图
- PowerPoint 适合课堂展示、汇报插图和可编辑图形

截图前需要检查账号、路径、Token 和其他敏感信息是否暴露。

## 十四、数据、实验与代码记录

### 1. 数据与实验工具

#### Jupyter Notebook

适合：

- 交互式数据检查
- 小规模分析实验
- 按顺序保留运行过程
- 记录环境和实验说明

#### TensorBoard

适合：

- 查看损失函数变化
- 比较评价指标
- 分析参数分布
- 对比训练过程
- 记录实验配置

#### Weights & Biases

适合：

- 集中跟踪多个实验
- 对比参数和指标
- 多人协作
- 管理实验结果

本地单次调试可以优先使用 Jupyter 或 TensorBoard。多实验协作需要确认权限和上传范围后，再考虑在线实验平台。

### 2. 代码与版本管理

科研代码管理流程包括：

1. 使用 VS Code 等工具查看、修改和运行代码
2. 使用 Git 记录代码、配置和文档的变化
3. 使用 GitHub 同步远程仓库
4. 使用 README、Issue 和提交记录开展协作

提交说明需要明确回答：

- 修改了什么
- 为什么修改
- 当前实验结果是什么

仓库中不能提交：

- 私有数据集
- 大型模型文件
- 密码
- 私钥
- Token
- 其他敏感信息

## 十五、智能体辅助科研的使用边界

智能体可以辅助完成：

- 扩展检索关键词
- 整理候选论文列表
- 检查文献记录字段是否完整
- 协助拆解连续任务

但以下信息必须回到原始来源核验：

- 论文题目、作者、年份和 DOI
- 正式发表来源和论文版本
- 论文内容与当前研究问题的相关性
- 实验结果是否可信
- 论文结论是否得到证据支持

智能体可以提高资料整理和任务执行效率，但科研事实、证据和最终判断仍然需要研究者负责。

## 十六、本次课程形成的文献调研流程

通过本次学习，我形成了以下文献调研流程：

1. 根据研究方向卡片明确研究对象、任务、方法和具体问题
2. 建立中英文关键词矩阵
3. 使用宽泛关键词寻找综述和研究入口
4. 逐步加入任务词、方法词和问题词缩小范围
5. 找到代表性论文并建立种子论文
6. 通过参考文献、被引用论文和相关论文扩展文献
7. 使用 Google Scholar 发现论文和引用关系
8. 使用 DBLP、DOI 和出版社页面核对书目信息与正式版本
9. 使用十分钟快速筛选法将论文分为精读、备查和排除
10. 使用 Zotero 完成分类、标签、PDF 关联和归档理由记录
11. 导出并核验 BibTeX 信息
12. 使用 Obsidian 记录论文笔记、检索过程和研究问题
13. 使用 Git 和 GitHub 管理笔记、代码与版本历史
14. 使用智能体辅助整理，但对关键信息返回原始来源核验

## 十七、学习总结

本次课程让我认识到，文献调研不是简单搜索论文，而是一套由研究问题定义、关键词设计、检索迭代、版本核对、引用追踪、快速筛选、文献管理和知识记录共同组成的研究流程。文献等级、引用量和智能工具只能提供辅助信息，真正决定论文价值的是其与研究问题的相关性、方法是否合理、证据是否充分以及是否具有复现价值。后续开展推荐系统研究时，我需要完整保留检索式、筛选理由、论文版本和阅读笔记，使文献调研过程更加规范、可核验和可复现。
